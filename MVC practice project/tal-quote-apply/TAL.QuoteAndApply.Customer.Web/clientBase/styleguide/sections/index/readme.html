Overview

This styleguide is a library of UI components for TAL Quote & Apply.

Atomic design principles

Atomic design is a UI design paradigm which involves building encapsulated, standalone components, and developing complexity largely by grouping and nesting simpler components. In practice, it involves the development of a library of standard components, much like this styleguide.

This library is hierarchical: the most basic components are called atoms, because they cannot be divided into simpler components. These for the basic building blocks of the UI, and include simple form fields, buttons, cards, and modals. These should be essentially immutable, their appearance and behavior not changing with their context, expect for a small and predefined set of parameters which form the components interface.

More complex molecules are built up of atomic subcomponents.

Organisms are the most complex elements, formed out of multiple molecules. These generally correspond to full views in the UI.

Diligently employing this paradigm can improve front end codebases in several key ways:

  • Scalability. New components are generally just collections of existing components, so minimal additional work or code is required to add views.
  • Consistency. Most UI elements across the project should be instances of a UI component from the styleguide, and hence have a well defined interface and form. They can easily be compared to the styleguide to judge whether they have been correctly used.
  • Testability. Due to the encapsulated nature of components, especially atomic ones, it's easy set up unit tests, markup tests, and and even visual regression tests for the styleguide components.
  • Clarity of design. Complexity in the UI arises largely out of explicit composition, with minimal case-specific code. This makes it easy to reason about the design and how it has been translated into code.

Local development process

grunt fedServe

This will make the styleguide available at localhost:8000, and will watch the project files for changes, rebuilding and updating the served site when they occur.

grunt fedTpl

This creates an Angular JS template cache from the current templates. Run this when new html files are added.

grunt fedInject

This injects the latest css and js assets into the styleguide. Run this when new css and js files are added.

Preview server

grunt fedDeploy

This will build the styleguide app and deploy it to the preview server specified in the GruntFile.js. The credentials for the ftp transfer are stored in the .ftppass file
